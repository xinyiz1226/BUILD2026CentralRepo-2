DTOs & Contracts

The project does not define explicit API DTO classes. Contract shapes are implicit in WebForms controls and HTML output: login flow consumes ReturnUrl, report refresh consumes date form fields, and report tables are emitted as HTML from DataTable-backed query results.

Communication Patterns

Communication is synchronous only: browser requests WebForms pages, and page handlers synchronously query SQL Server through SqlClient. The app redirects authentication requests to an external gateway (AuthGatewayLoginUrl/AuthGatewayLogoutUrl). No message broker, retries, circuit breaker policy, or service discovery mechanism is configured in this repository. Security posture is based on Forms Authentication cookies and authorization rules in web.config; HTTPS/TLS enforcement is not explicitly configured in project files.

Core Business Workflows

This application supports managers reviewing banking operational reports by authenticating users and presenting aggregated reporting views over loan and transaction data.

Primary Workflows

Workflow 1: Authenticate and Open Dashboard

  1. User requests dashboard page.
  2. If not authenticated, the app redirects to /Login.aspx and then to external auth gateway.
  3. After successful authentication, user returns and dashboard initializes date range.
  4. The app executes reporting queries and renders summary/grid output.

Workflow 2: Refresh Reports for Date Range

  1. User submits start and end dates from dashboard filter.
  2. RunReports validates the date range and rejects invalid ranges with status message.
  3. The app runs loan summary, delinquency, and daily transaction volume queries.
  4. Updated report sections are returned to the user.

Cross-Service Data Flows

Business data flow is primarily internal to one service with SQL-backed reads. The only cross-service interaction is authentication redirection to ZavaAuthGateway; reporting data composition itself is done inside ZavaReportDashboard by merging multiple SQL query results into a single rendered dashboard response.

Business Rules & Decision Logic

  • Access rule: unauthenticated users cannot view reports and are redirected to login/auth gateway.
  • Validation rule: report refresh requires parseable start/end dates and end >= start.
  • Reporting rule: dashboard combines three independent data sets (loan summary, delinquency, daily volume) into one manager view.
  • Session rule: authenticated identity controls UI sign-in status and login/logout link visibility.

Because dependencies are framework in-box assemblies, migration risk is concentrated in framework API compatibility and WebForms runtime replacement rather than external package upgrades.
