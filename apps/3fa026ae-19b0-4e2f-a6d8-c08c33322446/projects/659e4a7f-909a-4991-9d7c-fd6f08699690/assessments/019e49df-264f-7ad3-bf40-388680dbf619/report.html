Core Business Workflows

The service evaluates customer lending risk and allows operations teams to adjust model parameters that influence those decisions. Its primary workflows are risk scoring, risk factor retrieval, and model coefficient maintenance.

Primary Workflows

Workflow 1: Score Loan Application

  1. Entry: client invokes ScoreLoanApplication with applicationId.
  2. Service validates identifier and loads application, account balance, and latest credit score.
  3. Service loads model parameters and computes weighted score.
  4. Decision logic maps score to rating (A..F) and approval multiplier.
  5. Service writes a RiskAssessments record and returns score + max approved amount.

Workflow 2: Retrieve Customer Risk Factors

  1. Entry: client invokes GetRiskFactors with customerId.
  2. Service validates identifier and queries credit score, open exposure, average balance, and active accounts.
  3. Service loads active risk factor metadata and computes normalized factor values.
  4. Service calculates overall weighted score and returns factor list.

Workflow 3: Update Risk Model

  1. Entry: client invokes UpdateRiskModel with one or more parameter updates.
  2. Service validates payload and opens a DB transaction.
  3. For each parameter, service performs upsert behavior into RiskModelParameters.
  4. Transaction commits and service returns update summary.

Cross-Service Data Flows

No inter-service orchestration is defined in this repository. All business workflows are intra-service and data composition happens by issuing SQL queries across related banking tables within a shared SQL Server database.

Business Workflow Sequence

sequenceDiagram participant Analyst as "Loan Analyst" participant Service as "Risk Engine Service" participant DB as "SQL Server" Analyst->>Service: Submit score request for loan application Service->>DB: Fetch application, balance, latest credit score DB-->>Service: Loan context data Service->>DB: Fetch active model parameters DB-->>Service: Parameter set Service->>Service: Apply weighted scoring and rating rules alt Score maps to approved rating (A-E) Service->>Service: Calculate non-zero approval multiplier else Score maps to deny rating (F) Service->>Service: Set approval multiplier to zero end Service->>DB: Persist risk assessment outcome DB-->>Service: Persisted Service-->>Analyst: Return score, rating, max approved amount

Business Rules & Decision Logic

  • Input guard rules: applicationId and customerId must be greater than zero; model updates require at least one parameter.
  • Scoring rule: final score is derived from weighted normalized components (credit score, balance ratio, term, requested amount) plus a configurable base score.
  • Rating thresholds: A (>=85), B (>=75), C (>=65), D (>=55), E (>=45), F (<45).
  • Approval decision rule: multiplier by rating (A 1.25, B 1.10, C 0.90, D 0.70, E 0.50, F 0.00).
  • Factor scoring rule: individual factors are normalized/clamped and weighted to compute overall risk factor score.
  • Transaction rule: model parameter updates execute in a single SQL transaction for consistency.
